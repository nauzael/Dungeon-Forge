KEY IMPROVEMENT
─────────────────────────────────────────────────────────────────────────────────

BEFORE (Broken):
  Google Auth → URL Received (instant) → RELOAD (100ms) → Fail!
  Problem: Supabase hadn't processed tokens yet
  Result: Session empty → infinite loading ❌

AFTER (Fixed):
  Google Auth → URL Received (instant) → WAIT 2 SECONDS → RELOAD → Success!
  Benefit: Supabase processes tokens during the wait
  Result: Session established → user logged in ✅

This is the ONLY code change needed. Everything else is debugging infrastructure.

Common Failure Points & Quick Fixes:

  Missing: [OAuth] Deeplink received
    → Issue: Deeplink not reaching app
    → Fix: Check deeplink URL in Diagnostics
    → Should be: com.tupaquete.dndcompanion://login-callback

  Missing: [Auth] onAuthStateChange fired
    → Issue: Session not established
    → Fix: Check Supabase credentials in Diagnostics
    → Verify VITE_SUPABASE_URL and VITE_SUPABASE_ANON_KEY

  Appears: [Auth] No session found
    → Issue: Session establishment failed
    → Fix: Check Supabase OAuth provider settings
    → Verify redirect URI in Supabase dashboard
