ORIGINAL PROBLEM
================
User reported: "OAuth login se queda cargando eternamente"
(OAuth login hangs forever on Android)

THE FIX
=======
Changed timeout from 100ms → 2000ms in App.tsx line 92
Reason: Give Supabase time to process OAuth tokens before page reload

TECHNICAL DETAILS
=================

The fix works by:
  1. Receiving OAuth callback from Google (deeplink)
  2. Extracting access_token from URL
  3. WAITING 2 SECONDS ← This is the critical part
  4. During those 2 seconds:
     - Supabase receives the OAuth callback
     - Supabase validates the access_token
     - Supabase creates a session
     - Supabase stores session in database
  5. After 2 seconds, app reloads
  6. onAuthStateChange fires with VALID session
  7. App shows Character List
  8. User is logged in ✓

Without the 2-second wait (100ms original):
  - App reloads before Supabase finishes
  - Session is empty
  - onAuthStateChange fires with null session
  - App shows loading screen
  - User stuck forever ✗
