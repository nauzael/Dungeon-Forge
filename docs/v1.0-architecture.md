## 🔄 Flujo OTA

```
Developer:
  npm run build          →  dist/
  node scripts/build_ota.mjs  →  ota-release/app-update-{version}.zip
                               →  ota-release/version.json
                               →  Sube ambos a Supabase bucket "updates"

App (en dispositivo):
  App abre  →  fetch version.json  →  compara local vs remoto
  Si diferente  →  CapacitorUpdater.download()  →  set() + restart
```

## 🚨 Procedimiento de Rollback Completo

### Opción A: Rollback Solo OTA (rápido, <1 min)
```bash
# Devuelve la versión "live" en Supabase a v1.0 inmediatamente
node scripts/restore_ota.mjs 2026.4.16-123234 "Rollback de emergencia a v1.0"
```

### Opción B: Rollback Código + OTA (completo)
```bash
# 1. Revertir código local a v1.0
git checkout v1.0.0

# 2. Restaurar OTA
node scripts/restore_ota.mjs 2026.4.16-123234 "Rollback completo a v1.0"

# 3. Sincronizar dependencias
npm install
```

### Opción C: Modo Emergencia (rama archivada)
```bash
# La rama stable-v1.0 jamás se mueve del estado exacto del sellado
git checkout stable-v1.0
node scripts/restore_ota.mjs 2026.4.16-123234 "Rollback de emergencia"
```
