## 🏗️ 1. Arquitectura

### Patrón
SPA monolítica con ruteo por estado (`useState<ViewState>`) en `App.tsx`.
5 vistas: `list | create | sheet | dm-dashboard | observer-sheet`.

**Fortalezas:**
- ✅ Lazy loading en CreatorSteps, SheetTabs, DMDashboard
- ✅ Context API para temas/sync/diálogos
- ✅ Código splitting manual (vendor, firebase)
- ✅ Mobile-first + PWA + Capacitor

**Debilidades:**
- 🔴 **App.tsx: 1,444 líneas** — maneja auth, routing, CRUD, sync, OTA, import/export, themes
- 🔴 **utils/firebase.ts: 1,203 líneas** — todo Firebase en un solo archivo
- 🟡 **Dual directory**: `components/` y `src/components/` con imports mezclados
- 🟡 **Character interface**: 102 propiedades opcionales — monstruo de tipo único

### Sync Pipeline (complejidad alta)
```
User Action → setCharacters(fn) [prev spread] → dirty tracking (ref) 
  → debounce 500ms → localStorage → Firestore → RTDB broadcast
  → listener loopback guard (timestamp compare)
```
